## 一、已解决问题

以下问题在早期分析中发现，现已通过代码改进解决。

| # | 原问题 | 严重等级 | 解决方式 | 关键代码 |
|---|--------|---------|---------|---------|
| S1 | 领域实体形同虚设 | 严重 | 仓储接口全部返回领域实体类型，仓储实现使用 `to_domain()`/`from_domain()` 转换，应用服务操作领域实体 | `domain/repositories/` 返回 `UserEntity` 等；`infrastructure/repositories/` 实现 `to_domain()` |
| S2 | 仓储接口违反依赖倒置原则 | 中等 | 仓储接口返回领域实体类型，不再需要 `TYPE_CHECKING` 技巧 | `domain/repositories/` 返回类型已修正 |
| S3 | N+1 权限查询 | 中等 | `require_permission` 使用 `get_user_all_menus()` 单次 JOIN 查询；`get_users_roles_batch()` 批量获取角色 | `api/dependencies/permission.py`；`infrastructure/repositories/menu_repository.py` |
| S4 | Redis 已配置但未集成 | 中等 | `CacheService` 实现 Token 黑名单、用户信息缓存、权限缓存，支持 Redis 降级 | `infrastructure/cache/cache_service.py` |
| S5 | Token 黑名单缺失 | 严重 | `AuthService.logout()` 加入黑名单，`get_current_user_id()` 检查黑名单 | `application/services/auth_service.py`；`api/dependencies/auth.py` |
| S6 | IP 过滤未启用 | 中等 | `IPFilterMiddleware` 已注册，`IPFilterCache` 实现规则缓存和自动刷新 | `infrastructure/http/ip_filter_cache.py`；`infrastructure/http/ip_filter_middleware.py` |
| S7 | 批量删除逐条执行 | 中等 | `UserRepository.batch_delete()` 使用 `WHERE id IN(...)` 单次 SQL | `infrastructure/repositories/user_repository.py` |

## 二、新发现问题

以下问题在本次代码分析中新发现，早期文档未提及。

| # | 问题 | 严重等级 | 影响范围 | 位置 | 建议修复方式 |
|---|------|---------|---------|------|------------|
| N1 | 应用层直接依赖基础设施层 | 中等 | DDD 分层约束 | `IPRuleService` 导入 `from src.infrastructure.http.ip_filter_cache import get_ip_filter_cache`；`MenuService`/`UserService` 导入 `from src.infrastructure.cache.cache_service import CacheService` | 在领域层定义 `CachePort`/`IPFilterPort` 抽象接口，基础设施层实现，应用层通过抽象调用 |
| N2 | IPRuleEntity 时区不一致 | 轻微 | IP 规则过期判断 | `domain/entities/ip_rule.py` `is_expired` 使用 `datetime.now()`（无时区），`infrastructure/repositories/ip_rule_repository.py` `get_effective_ip_rules()` 使用 `datetime.now(timezone.utc)` | 统一使用 `datetime.now(timezone.utc)` |
| N3 | 实体行为方法未被充分使用 | 轻微 | 状态变更封装 | `UserEntity.activate()`/`deactivate()`、`RoleEntity.activate()`/`deactivate()` 已定义但应用层未调用，直接通过 `update_info()` 修改 `is_active` | 应用服务调用实体行为方法替代直接修改字段 |
| N4 | MenuService 访问私有属性 | 轻微 | 封装性 | `application/services/menu_service.py` 第249行 `menu._meta = meta_entity` | 使用构造函数或 `with_meta()` 方法 |
| N5 | SystemLogEntity 向后兼容别名 | 轻微 | 代码清晰度 | `domain/entities/log.py` 第143行 `SystemLogEntity = OperationLogEntity`，注释"后续阶段3会逐步清理" | 统一使用 `OperationLogEntity`，移除别名 |
| N6 | 数据库表名不一致 | 轻微 | 文档与代码对齐 | 文档中使用 `sys_system_logs`，实际表名是 `sys_operation_logs`（OperationLogEntity 对应 SystemLog 模型） | 统一文档和代码中的表名 |
