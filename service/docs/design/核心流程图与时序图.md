## 1. 认证流程时序图

### 1.1 登录流程

```mermaid
sequenceDiagram
    participant C as 客户端
    participant R as AuthRouter
    participant D as Depends(get_auth_service)
    participant S as AuthService
    participant PS as PasswordService
    participant TS as TokenService
    participant UR as UserRepository
    participant RR as RoleRepository
    participant MR as MenuRepository
    participant CS as CacheService
    participant DB as Database

    C->>R: POST /api/system/login<br/>{username, password}
    R->>D: 获取 AuthService 实例
    D->>UR: 创建 UserRepository(session)
    D->>RR: 创建 RoleRepository(session)
    D->>MR: 创建 MenuRepository(session)
    D->>TS: 创建 TokenService(settings)
    D->>CS: 创建 CacheService(redis)
    D->>S: 创建 AuthService(session, repos, services)

    R->>S: login(LoginDTO)
    S->>CS: get_user_info(username)
    CS-->>S: None (缓存未命中)
    S->>UR: get_by_username(username)
    UR->>DB: SELECT * FROM sys_users WHERE username=?
    DB-->>UR: User ORM 对象
    UR->>UR: to_domain() 转换
    UR-->>S: UserEntity 对象
    S->>CS: set_user_info(username, user)

    S->>PS: verify_password(input, hash)
    PS-->>S: True/False

    alt 密码错误
        S-->>R: raise UnauthorizedError
        R-->>C: 401 {code: 401, message: "用户名或密码错误"}
    end

    S->>TS: create_access_token(user_id, username)
    TS-->>S: access_token
    S->>TS: create_refresh_token(user_id, username)
    TS-->>S: refresh_token

    S->>CS: get_user_menus(user_id)
    CS-->>S: None (缓存未命中)
    S->>RR: get_user_roles(user_id)
    RR->>DB: JOIN sys_userinfo_roles + sys_roles
    DB-->>RR: RoleEntity 列表
    RR-->>S: roles
    S->>MR: get_user_all_menus(user_id)
    MR->>DB: 单次 JOIN 查询<br/>用户→角色→菜单
    DB-->>MR: MenuEntity 列表
    MR-->>S: menus
    S->>CS: set_user_menus(user_id, menus)

    S-->>R: {accessToken, refreshToken, roles, permissions}
    R-->>C: 200 {code: 0, data: {...}}
```

### 1.2 Token 刷新流程

```mermaid
sequenceDiagram
    participant C as 客户端
    participant R as AuthRouter
    participant S as AuthService
    participant TS as TokenService
    participant UR as UserRepository
    participant RR as RoleRepository
    participant MR as MenuRepository

    C->>R: POST /api/system/refresh-token<br/>{refreshToken}
    R->>S: refresh_token(refresh_token)

    S->>TS: decode_token(refresh_token)
    TS-->>S: payload

    S->>TS: verify_token_type(payload, "refresh")
    alt token 类型不匹配
        S-->>C: 401 {message: "无效的刷新令牌"}
    end

    S->>UR: get_by_id(user_id)
    UR-->>S: user

    alt 用户不存在或不活跃
        S-->>C: 401 {message: "用户不可用"}
    end

    S->>TS: create_access_token(user_id, username)
    TS-->>S: new_access_token
    S->>TS: create_refresh_token(user_id, username)
    TS-->>S: new_refresh_token

    S->>RR: get_user_roles(user_id)
    S->>MR: get_role_menu_ids(role_ids)

    S-->>C: 200 {code: 0, data: {accessToken, refreshToken, ...}}
```

## 2. RBAC 权限校验时序图

```mermaid
sequenceDiagram
    participant C as 客户端
    participant R as 路由端点
    participant P as require_permission("user:delete")
    participant AUTH as get_current_user_id
    participant TS as TokenService
    participant CS as CacheService
    participant UR as UserRepository
    participant MR as MenuRepository
    participant DB as Database

    C->>R: DELETE /api/system/user/{id}<br/>Authorization: Bearer <token>

    R->>P: Depends(require_permission)
    P->>AUTH: Depends(get_current_user_id)

    AUTH->>TS: decode_token(access_token)
    TS-->>AUTH: payload {sub: user_id, type: "access"}
    AUTH->>TS: verify_token_type(payload, "access")

    AUTH->>CS: is_token_blacklisted(jti)
    CS-->>AUTH: False (未在黑名单)

    AUTH-->>P: user_id

    P->>CS: get_user_info(user_id)
    CS-->>P: UserEntity (缓存命中) / None

    alt 缓存未命中
        P->>UR: get_by_id(user_id)
        UR->>DB: SELECT FROM sys_users
        DB-->>UR: User ORM
        UR->>UR: to_domain()
        UR-->>P: UserEntity
        P->>CS: set_user_info(user_id, user)
    end

    alt is_superuser == 1
        P-->>R: 超级用户，跳过权限检查 ✅
    else 普通用户
        P->>CS: get_user_menus(user_id)
        CS-->>P: menus (缓存命中) / None

        alt 缓存未命中
            P->>MR: get_user_all_menus(user_id)
            MR->>DB: 单次 JOIN 查询<br/>用户→角色→菜单
            DB-->>MR: MenuEntity 列表
            MR-->>P: menus
            P->>CS: set_user_menus(user_id, menus)
        end

        alt menus 中存在 name=="user:delete" 且 menu_type==2
            P-->>R: 权限校验通过 ✅
        else 无对应权限
            P-->>C: 403 {code: 403, message: "权限不足"}
        end
    end

    R->>R: 执行业务逻辑
    R-->>C: 200 {code: 0, data: ...}
```

## 3. 动态路由加载时序图

```mermaid
sequenceDiagram
    participant FE as Vue3 前端
    participant R as AuthRouter
    participant S as AuthService
    participant MR as MenuRepository
    participant MMR as MenuMetaRepository
    participant DB as Database

    FE->>R: GET /api/system/get-async-routes<br/>Authorization: Bearer <token>
    R->>S: get_async_routes(user_id)

    S->>MR: get_user_menus(user_id)
    MR->>DB: 查询用户角色关联的所有菜单<br/>(menu_type IN [0, 1])
    DB-->>MR: Menu 列表
    MR-->>S: menus

    S->>S: _build_route_tree(menus)<br/>递归构建树形结构

    loop 对每个菜单项
        S->>MMR: 获取 meta 信息
        S->>S: 组装 route 对象<br/>{path, name, component, meta}
    end

    S-->>R: 路由树
    R-->>FE: 200 {code: 0, data: [routeTree]}

    FE->>FE: permission store 解析路由树<br/>动态添加到 Vue Router
```

## 4. 请求处理全流程图

```mermaid
flowchart TD
    A[客户端请求] --> B{CORS 检查}
    B -->|不通过| C[403 CORS 错误]
    B -->|通过| D[RequestLoggingMiddleware<br/>记录请求日志]

    D --> E{需要认证?}
    E -->|公开接口| F[直接进入路由处理]
    E -->|受保护接口| G[get_current_user_id<br/>解码 JWT Token]

    G --> H{Token 有效?}
    H -->|无效/过期| I[401 Unauthorized]
    H -->|有效| H2{Token 在黑名单?}

    H2 -->|在黑名单| I2[401 Token 已注销]
    H2 -->|不在黑名单| J{需要权限?}

    J -->|不需要| F
    J -->|需要| K[require_permission<br/>缓存优先 + 单次 JOIN 查询]

    K --> L{权限足够?}
    L -->|不足| M[403 Forbidden]
    L -->|足够| F

    F --> N[调用应用服务]
    N --> O[应用服务编排业务逻辑<br/>操作领域实体]
    O --> P[调用仓储接口]
    P --> Q[仓储实现<br/>from_domain/to_domain 转换]
    Q --> R{操作成功?}

    R -->|成功| S[get_db 自动 commit]
    R -->|失败| T[get_db 自动 rollback]

    S --> U[统一响应 success_response]
    T --> V{异常类型}

    V -->|AppError| W[ExceptionHandler<br/>返回业务错误]
    V -->|ValidationError| X[422 参数校验错误]
    V -->|Exception| Y[500 内部错误]

    U --> Z[返回响应给客户端]
    W --> Z
    X --> Z
    Y --> Z

    D -.->|非 GET 请求| AA[写入操作审计日志<br/>sys_operation_logs]
```

## 6. 依赖注入装配流程图

```mermaid
flowchart TD
    subgraph 请求进入
        REQ[HTTP 请求] --> ROUTE[路由端点]
    end

    subgraph Depends 链
        ROUTE --> GET_DB[Depends get_db]
        ROUTE --> GET_SVC[Depends get_xxx_service]

        GET_DB --> SESSION[AsyncSession<br/>auto commit/rollback]

        GET_SVC --> TOKEN_SVC[Depends get_token_service<br/>new TokenService settings]
        GET_SVC --> PWD_SVC[Depends get_password_service<br/>PasswordService 静态]
        GET_SVC --> CACHE_SVC[Depends get_cache_service<br/>CacheService redis]
        GET_SVC --> REPO[Depends get_xxx_repository<br/>new XxxRepository session]

        REPO --> SESSION
    end

    subgraph 服务实例
        GET_SVC --> SVC[XxxService<br/>session + repos + domain_services + cache]
    end

    subgraph 权限校验
        ROUTE --> PERM[Depends require_permission]
        PERM --> CURR_USER[Depends get_current_user_id<br/>TokenService.decode + 黑名单检查]
        PERM --> CACHE[CacheService<br/>优先缓存查询]
        PERM --> MENU_REPO[MenuRepository.get_user_all_menus<br/>单次 JOIN 查询]

        CACHE --> SESSION
        MENU_REPO --> SESSION
    end

    SVC --> EXEC[执行业务逻辑]
    PERM --> EXEC
```

## 7. 异常处理流程图

```mermaid
flowchart TD
    REQ[请求处理] --> RESULT{结果}

    RESULT -->|正常返回| OK[success_response<br/>code: 0]
    RESULT -->|AppError 子类| APP_ERR{异常类型}

    APP_ERR -->|NotFoundError| R404[404 code: 404]
    APP_ERR -->|ConflictError| R409[409 code: 409]
    APP_ERR -->|UnauthorizedError| R401[401 code: 401]
    APP_ERR -->|ForbiddenError| R403[403 code: 403]
    APP_ERR -->|ValidationError| R422[422 code: 422]
    APP_ERR -->|RateLimitError| R429[429 code: 429]
    APP_ERR -->|BusinessError| R400[400 code: 400]

    RESULT -->|RequestValidationError| V422[422 参数校验失败<br/>detail: 字段错误列表]
    RESULT -->|未知 Exception| R500[500 内部服务器错误<br/>code: 500]

    OK --> RES[统一响应格式<br/>code + message + data]
    R404 & R409 & R401 & R403 & R422 & R429 & R400 --> RES
    V422 --> RES
    R500 --> RES
```
