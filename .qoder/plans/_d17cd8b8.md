# 前后端服务联调 - 后端 API 全面适配

## 核心原则

前端页面不做修改。以 `web/mock/` 下的 Mock 数据为"参考标准"，后端 API 必须产出与 Mock 完全一致的响应格式。需要修改的仅有：
- 后端所有 API 路由和响应格式
- 前端 HTTP 客户端 `baseURL` 配置（基础设施配置，非页面修改）
- 前端 Mock 开关（连接真实后端时需关闭）

## 问题清单

### 响应格式问题
| 问题 | 后端当前 | Mock 标准 |
|------|---------|----------|
| 成功码 | `code: 200` | `code: 0` |
| 登录响应 | 嵌套 `userInfo: {}` 结构 | 扁平结构 `avatar/username/nickname` 直接在 data 下 |
| expires 格式 | 秒数 `1800` | 日期字符串 `"2030/10/30 00:00:00"` |
| 分页响应 | `{ total, pageNum, pageSize, totalPage, rows }` | `{ list, total, pageSize, currentPage }` |

### 路由缺失/不匹配
| 前端调用路径 | 方法 | 后端当前状态 | 所属模块 |
|-------------|------|------------|---------|
| `/refresh-token` | POST | `/refresh` (路径不同) | 认证 |
| `/mine` | GET | `/user/info` (路径不同) | 个人信息 |
| `/mine-logs` | GET | 不存在 | 个人信息 |
| `/get-async-routes` | GET | 不存在 | 动态路由 |
| `/user` | POST | `/user/list` POST (路径不同) | 用户管理 |
| `/role` | POST | `/role/list` GET (路径+方法不同) | 角色管理 |
| `/menu` | POST | `/menu/tree` GET (路径+方法不同) | 菜单管理 |
| `/list-all-role` | GET | 不存在 | 用户管理辅助 |
| `/list-role-ids` | POST | 不存在 | 用户管理辅助 |
| `/role-menu` | POST | 不存在 | 角色管理辅助 |
| `/role-menu-ids` | POST | 不存在 | 角色管理辅助 |
| `/dept` | POST | 不存在 | 部门管理 |
| `/online-logs` | POST | 不存在 | 系统监控 |
| `/login-logs` | POST | 不存在 | 系统监控 |
| `/operation-logs` | POST | 不存在 | 系统监控 |
| `/system-logs` | POST | 不存在 | 系统监控 |
| `/system-logs-detail` | POST | 不存在 | 系统监控 |
