## 核心功能

- **阶段一：数据库初始化验证**
  - 清空 SQLite 测试数据库（删除 `service/sql/test.db`）
  - 设置 `APP_ENV=testing` 使用 `.env.testing` 配置（SQLite: `sql/test.db`）
  - 执行 `python -m scripts.cli initdb` 验证建表流程
  - 执行 `python -m scripts.cli seedrbac` 验证 RBAC 种子数据初始化
  - 执行 `python -m scripts.cli seeddata` 验证测试数据初始化
  - 执行 `python -m scripts.cli createsuperuser` 验证超级管理员创建
  - 或一键执行 `python -m scripts.cli initall` 验证完整初始化链路
  - 确认数据库表结构和约束正确（外键、唯一索引等）
  - 确认种子数据完整性（菜单树、角色-菜单关联、用户-角色关联）
  - 验证数据库模型完整覆盖：`sys_users`、`sys_roles`、`sys_menus`、`sys_menumeta`、`sys_departments`、`sys_ip_rules`、`sys_systemconfig`、`sys_loginlog`、`sys_logs`、`sys_userinfo_roles`、`sys_userrole_menu`

## 技术栈

- 后端: FastAPI + SQLModel + SQLAlchemy 2.0 (AsyncIO) + classy-fastapi + pytest
- 前端: Vue 3 + TypeScript + Element Plus + Pure Admin + Pinia
- 测试工具: pytest (后端), Playwright MCP (前端UI测试)
- HTTP客户端: Axios（前端 `PureHttp` 封装类，baseURL=/api/system）
- 数据库: SQLite（测试环境，`.env.testing`）/ SQLite（开发环境，`.env.development`） / MySQL（可选生产环境）
- 缓存: Redis（CacheService，降级为安全默认值）/ 内存缓存（IPFilterCache）
- 认证: JWT 双令牌（accessToken 30min + refreshToken 7天）+ HTTPBearer
- 权限: RBAC（Menu.menu_type=2 实现按钮级权限，`require_permission(code)` 依赖工厂）
- 限流: slowapi + Redis/Memory 存储（Limiter 中间件）

### 关键技术决策

- 从数据库初始化开始验证，确保不遗漏任何建表/种子数据场景
- 后端测试优先于前端测试，确保 API 正确后再进行 UI 测试
- **后端测试必须使用 `.env.testing` 配置**（`APP_ENV=testing`），使用 SQLite 内存数据库，不依赖外部服务
- 前端联调时后端使用 `.env.development` 配置（SQLite: `sql/dev.db`），需确保后端服务可用
- 使用 Playwright MCP 进行 UI 自动化测试，而非纯手动操作，确保测试可复现
- 问题修复采用"发现一批、修复一批、验证一批"的迭代方式
- SQLite 测试数据库在初始化验证后需保留，供前端联调时后端使用
- 联调验证全部通过后，进行前端生产打包验证，确保整个开发到发布流程闭环
- 后端所有路由使用 `classy-fastapi.Routable` 类风格定义，统一挂载在 `API_SYSTEM_PREFIX`（`/api/system`）下
- 前端 API 层使用 `BaseApi` 基类封装标准 CRUD，子模块继承后按需扩展
- 权限控制通过 `require_permission(code)` 依赖工厂实现，code 对应 `Menu.name`（menu_type=2）
- 认证流程：HTTPBearer → `get_current_user_id()` 解析 JWT → `get_current_active_user()` 获取用户信息（优先缓存）
- 应用中间件栈：CORS → RequestLoggingMiddleware（请求日志）→ IPFilterMiddleware（IP黑白名单过滤）→ Limiter（限流）
- 限流器（Limiter）使用 slowapi，支持 Redis/Memory 双存储模式，白名单 IP 可跳过限流

## 前后端 API 对接验证要点

### 登录流程
1. 前端 `getLogin()` → POST `/api/system/login` → 返回 `{accessToken, refreshToken, expires}`
2. 前端 `setToken(data)` → 存储到 localStorage
3. 前端 `getAsyncRoutes()` → GET `/api/system/get-async-routes` → 返回菜单路由树
4. 前端 `permission.ts` store → 构建动态路由 → `router.addRoute()`

### Token 刷新流程
1. 前端 HTTP 拦截器检测 token 过期（`expired = parseInt(data.expires) - now <= 0`）
2. 调用 `useUserStoreHook().handRefreshToken({refreshToken})` → POST `/api/system/refresh-token`
3. 刷新成功：更新 Authorization header，重放暂存请求
4. 刷新失败：调用 `logOut()` 清空状态跳转登录页

### 401 处理流程
1. 响应拦截器检测 status === 401
2. 排除白名单（`/refresh-token`, `/login`）
3. 检查 `isLoggingOut` 标志位防止并发登出
4. 显示过期提示 → 调用 `logOut()` → 跳转登录页
