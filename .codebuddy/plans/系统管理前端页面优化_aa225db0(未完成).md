## 产品概述

对系统管理模块前端8个子页面进行全面优化，统一表单交互方式、丰富表格数据展示、优化表单布局，最大化利用已封装组件。

## 核心功能

- 将 config 和 ip-rule 页面的 el-dialog 内联表单迁移为 ReDialog 函数式表单（addDialog），与其他5个页面保持一致
- 为所有表格补充后端接口返回但前端未展示的字段列，让数据展示更完整
- 统一所有表单页面布局，使用 ReCol 两列栅格布局；role 表单从单列改为两列
- 充分利用已封装组件：ReDialog、ReCol、ReSegmented、ReText、ReTreeLine、usePublicHooks、公共常量等

## 技术栈

- 前端框架：Vue3 + TypeScript（现有项目架构）
- 组件库：Element Plus + Pure Admin 封装组件
- 函数式弹窗/抽屉：addDialog / addDrawer（@/components/ReDialog、@/components/ReDrawer）
- 栅格布局：ReCol（封装 el-col）
- 分段控制器：ReSegmented
- 文本省略：ReText
- 树形连接线：ReTreeLine
- 公共样式：usePublicHooks（switchStyle、tagStyle）
- 公共常量：MenuChoices、GenderChoices、IPRuleTypeChoices、DictStatusChoices 等

## 实现方案

### 关键技术决策

1. **ReDialog 函数式表单迁移**：config 和 ip-rule 页面从 `el-dialog + dialogVisible + ruleForm` 模式迁移到 `addDialog + 独立 form.vue` 组件模式。遵循现有5个页面的成熟模式：hook.tsx 中调用 addDialog 传入 contentRenderer 渲染表单组件，表单组件通过 props 接收 formInline 数据，通过 defineExpose 暴露 getRef 方法供 beforeSure 回调中做表单校验。

2. **封装组件最大化利用**：

- **ReSegmented**：替代搜索表单中的状态 el-select（如用户/角色/部门/字典页面的 isActive 筛选），以及表单中的布尔选择（如 config 的 isActive/inherit、ip-rule 的 ruleType）
- **ReText**：用于表格长文本列（description、value 等）的省略展示，替代纯 showOverflowTooltip
- **ReTreeLine**：用于菜单树形表格和权限树的连接线展示，提升树形结构可读性
- **usePublicHooks（switchStyle/tagStyle）**：统一状态列的 switch 和 tag 样式，确保深色/浅色主题一致
- **公共常量（constants.ts）**：搜索表单的 el-option 选项使用 GenderChoices、IPRuleTypeChoices、DictStatusChoices 等常量，替代硬编码
- **render.tsx 公共渲染函数**：表格列中使用 renderMenuTypeTag、renderSwitch 等

3. **表格列补全原则**：展示所有有业务价值的接口返回字段。updatedTime 统一添加；description 有值时展示；email/phone 等已有但未展示的字段补上。长文本列使用 ReText 或 showOverflowTooltip。

4. **表单布局统一**：所有表单组件统一使用 `el-row + ReCol` 两列栅格布局（:value="12" :xs="24" :sm="24"），长文本字段使用全宽（默认 :value="24"）。label-width 统一 82px。

### 性能与可靠性

- 表格新增列对渲染性能无明显影响，数据量在分页控制下
- addDialog 模式下弹窗生命周期由 ReDialog 统一管理，内存释放更可靠
- 表单校验在 beforeSure 回调中完成，校验失败不关闭弹窗

### 实现注意事项

- config 和 ip-rule 新建 form.vue 严格遵循现有模式（props FormProps + ref + getRef + defineExpose）
- addDialog 的 contentRenderer 使用 `() => h(editForm, { ref: formRef, formInline: null })` 模式
- beforeSure 中通过 `formRef.value.getRef()` 获取表单实例进行 validate 校验
- 表格列 dayjs 格式化统一使用 `YYYY-MM-DD HH:mm:ss` 格式
- 新增 updatedTime 列使用 `formatter` 而非 `cellRenderer`，与现有 createdTime 列保持一致
- config 使用直接导入的 API 函数（getConfigList 等），ip-rule 也使用直接导入函数，改造时保持各自的调用方式不变
- ReText 用于 cellRenderer 中包裹长文本，而非直接替换 showOverflowTooltip 属性

## 架构设计

### 改造前后对比

**改造前（config/ip-rule）**：

```
index.vue → hook.tsx (dialogVisible + ruleForm) → index.vue (el-dialog 内联表单)
```

**改造后（config/ip-rule，与 user/role 等一致）**：

```
index.vue → hook.tsx (addDialog) → form.vue (独立表单组件 + ReCol 两列布局)
```

### 封装组件使用映射

| 组件 | 使用页面 | 具体用途 |
| --- | --- | --- |
| ReDialog (addDialog) | config、ip-rule | 函数式弹窗，替代 el-dialog |
| ReCol | config form、ip-rule form、role form | 两列栅格布局 |
| ReSegmented | config form（isActive/inherit）、ip-rule form（ruleType） | 布尔/枚举选择 |
| ReText | user/dept/config 表格 | 长文本列省略+Tooltip |
| ReTreeLine | menu/dept 表格、permission 权限树 | 树形连接线 |
| usePublicHooks (switchStyle/tagStyle) | config/ip-rule/role 表格和表单 | 统一 switch/tag 样式 |
| 公共常量 (GenderChoices等) | user/dictionary/ip-rule | 替代硬编码选项 |
| renderMenuTypeTag | permission 权限树 | 菜单类型标签渲染 |
