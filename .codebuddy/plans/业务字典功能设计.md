## 产品概述

系统引入 sys_dictionary 业务字典表，实现字典数据的增删改查管理，支持树形层级结构（parent_id），供前端下拉选择、标签映射等场景使用。

## 核心功能

- 后端：完整 DDD 分层实现字典 CRUD 接口（列表/详情/新增/编辑/删除/状态切换），包含按字典名称查询字典项的专用接口
- 前端：字典管理页面，支持树形展示、搜索筛选、新增/编辑弹窗、删除确认、状态切换
- RBAC：在种子数据中添加字典管理的菜单和权限项
- 联调：前后端接口对接，确保数据流通和交互正常

## 技术栈

- 后端：FastAPI + SQLModel + classy-fastapi + FastCRUD（沿用现有技术栈）
- 前端：Vue3 + TypeScript + Element Plus + @pureadmin/table（沿用现有技术栈）
- 数据库：MySQL，表 sys_dictionary 已在 SQL 中定义

## 实现方案

按照现有 DDD 分层模式，参照 department（部门管理）的树形结构实现字典管理模块。字典表具有 parent_id 字段，与部门管理类似的树形展示需求。

### 后端实现策略

1. **ORM 模型层**：创建 Dictionary(SQLModel)，包含 to_domain/from_domain 转换方法，注意不使用 `from __future__ import annotations`
2. **领域实体层**：创建 DictionaryEntity(@dataclass)，纯数据载体
3. **抽象仓储层**：创建 DictionaryRepositoryInterface(ABC)，定义 get_by_id/get_all/get_by_name/get_max_sort/create/update/delete 等抽象方法
4. **具体仓储层**：创建 DictionaryRepository，注入 AsyncSession，使用 FastCRUD 实现标准 CRUD；get_all 按 sort ASC 排序返回；新增 get_max_sort(parent_id) 方法查询同级最大排序值
5. **DTO 层**：创建 DictionaryCreateDTO/UpdateDTO/ResponseDTO/ListQueryDTO，字段 camelCase 命名
6. **应用服务层**：创建 DictionaryService，注入 session + repo，编排业务逻辑；新增时若 sort 未指定则自动计算同级最大值+1
7. **API 路由层**：创建 DictionaryRouter(Routable)，注册到 system_router，prefix="/dictionary"
8. **依赖注入**：创建 get_dictionary_service 工厂函数
9. **RBAC 种子数据**：在 rbac_defaults.py 中添加字典管理菜单和权限

### 前端实现策略

1. **API 层**：创建 DictionaryApi 继承 BaseApi，super("/dictionary")，覆写 list() 返回 Result（树结构不分页），添加 getByName 方法
2. **视图层**：4 文件标准结构（types.ts/rule.ts/hook.tsx/form.vue/index.vue），参照 dept 模块的树形展示模式
3. **常量**：在 constants.ts 中添加字典状态选项

### 数据流

```
前端 DictionaryApi → /api/system/dictionary/* → DictionaryRouter
→ Depends(get_dictionary_service) → DictionaryService(session, dict_repo)
→ DictionaryRepository(session) → Dictionary(SQLModel) → sys_dictionary 表
```

## 实现要点

- 字典表有 parent_id 字段支持树形层级，前端使用 handleTree() 将扁平数据转树，与部门管理一致
- 后端 list 接口返回全量数据（不分页），由前端构建树形展示；同时提供按 name 查询的接口供其他业务场景获取字典值
- ORM 模型 id 字段使用 uuid.uuid4().hex 保持与现有模型一致（32位），尽管 SQL 定义为 varchar(255)
- 时间字段使用 `sa_column=Column(DateTime(6), server_default=func.now())` 模式
- DTO 字段使用 camelCase（如 parentId, createdTime），与前端约定一致
- RBAC 权限：dictionary:view / dictionary:add / dictionary:edit / dictionary:delete
- 代码注释全部使用中文
- **sort 排序机制**：字典项按 sort 字段升序排列，后端 get_all 查询自动 ORDER BY sort ASC；新增字典项时，若未指定 sort 值，则自动计算同级（相同 parent_id）下已有最大 sort 值 +1 作为默认排序值；sort 字段支持手动修改
- **sort 自动递增**：在 DictionaryService.create_dictionary() 中，若 DTO 中 sort 为 None 或 0，则调用 dict_repo.get_max_sort(parent_id) 获取同级最大 sort 值后 +1；仓储层新增 get_max_sort(parent_id) 方法实现此查询
- **前端 sort 处理**：新增弹窗中 sort 字段默认不填（后端自动计算），但允许用户手动输入覆盖；编辑弹窗中 sort 字段显示当前值并支持修改

## 设计风格

沿用系统管理模块统一风格，与部门管理页面结构一致。左侧或顶部搜索区域，主体区域为树形表格展示，操作列包含编辑、新增子项、删除按钮。

## 页面规划

### 字典管理页面 (views/system/dictionary/index.vue)

- **搜索区域**：字典名称输入框 + 状态下拉选择 + 搜索/重置按钮
- **操作栏**：新增字典按钮，表格列配置、刷新、全屏切换
- **表格区域**：树形表格，列包含：字典名称(name)、显示标签(label)、字典值(value)、状态(el-tag)、排序(sort)、创建时间、描述、操作列(编辑/新增子项/删除)
- **弹窗表单**：新增/编辑字典项，字段包含：上级字典(级联选择)、字典名称、显示标签、字典值、状态(开关)、排序(数字输入，新增时默认自动递增，可手动修改)、描述(文本域)
