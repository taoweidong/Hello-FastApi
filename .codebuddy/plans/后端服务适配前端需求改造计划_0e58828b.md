## 产品概述

前端已有系统管理（用户/角色/菜单/部门）和系统监控页面，需要后端提供真实数据接口，数据持久化存储到数据库。

## 核心功能

- **部门管理**：部门树形结构CRUD，与用户关联
- **系统监控**：在线用户、登录日志、操作日志、系统日志的查询与管理
- **角色菜单权限**：角色与菜单的关联配置

## 前端已有页面

- 系统管理：用户管理、角色管理、菜单管理、部门管理
- 系统监控：在线用户、登录日志、操作日志、系统日志

## 后端现状

- 认证逻辑已适配完成
- 用户、角色、权限、菜单基础CRUD已实现
- 部门和日志接口目前返回stub数据，需要接入真实数据库

## 技术栈

- **后端框架**：FastAPI
- **ORM**：SQLModel + SQLAlchemy 2.0 (AsyncIO)
- **数据库**：SQLite（开发）/ PostgreSQL（生产）
- **认证**：JWT（已完成）
- **架构**：DDD分层架构（API → Application → Domain → Infrastructure）

## 实现策略

遵循最小改动原则，复用现有代码结构：

1. 在现有 `models.py` 中新增缺失的数据模型
2. 按现有模式新增对应的 Repository、Service、DTO
3. 将 `system_routes.py` 中的 stub 实现替换为真实数据库操作
4. 完善 Role-Menu 关联功能

## 数据模型设计

### 新增模型（基于现有架构）

```
Department      # 部门（树形结构）
├── id, name, parentId, sort, principal, phone, email, status, remark, createTime

LoginLog        # 登录日志
├── id, username, ip, address, system, browser, status, behavior, loginTime

OperationLog    # 操作日志
├── id, username, ip, address, system, browser, status, summary, module, operatingTime

SystemLog       # 系统日志
├── id, level, module, url, method, ip, address, system, browser, takesTime, requestTime, requestBody, responseBody

RoleMenuLink    # 角色-菜单关联表
├── role_id, menu_id
```

## API 接口清单

### 部门管理

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | /dept | 部门列表（扁平结构） |
| POST | /dept/create | 创建部门 |
| PUT | /dept/{id} | 更新部门 |
| DELETE | /dept/{id} | 删除部门 |


### 系统监控

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | /online-logs | 在线用户列表 |
| POST | /login-logs | 登录日志列表 |
| POST | /operation-logs | 操作日志列表 |
| POST | /system-logs | 系统日志列表 |
| POST | /system-logs-detail | 系统日志详情 |
| DELETE | /login-logs/batch | 批量删除登录日志 |
| DELETE | /operation-logs/batch | 批量删除操作日志 |
| DELETE | /system-logs/batch | 批量删除系统日志 |


### 角色菜单权限

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | /role-menu | 获取菜单权限树 |
| POST | /role-menu-ids | 获取角色已分配的菜单ID |
| POST | /role/{id}/menu | 保存角色菜单权限 |
