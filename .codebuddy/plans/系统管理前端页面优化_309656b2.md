## 产品概述

对系统管理模块（8个页面）和系统监控模块（4个页面）共12个前端页面进行全面优化，其中**业务字典页面需按目标图片重构为左右分栏主从布局**。核心改造包括：统一ReDialog函数式表单、修复monitor模块字段映射错误、丰富表格数据展示、优化表单布局、充分利用已封装组件。

## 核心功能

### System 模块（8页）

- **config / ip-rule**：el-dialog 内联表单迁移为 ReDialog 函数式表单（addDialog + 独立form.vue + ReCol两列）
- **role**：表单从单列改为 ReCol 两列布局，补 isActive 字段
- **user**：表格新增 email(ReText省略)、description(ReText省略)、updatedTime 列
- **menu**：表格新增 isActive(el-switch+switchStyle)、updatedTime 列，使用 ReTreeLine 增强树形连接线
- **permission**：角色列表新增 description 列，权限树用 renderMenuTypeTag + ReTreeLine
- **dept**：表格新增 code、principal、phone、email(ReText)、updatedTime 列，使用 ReTreeLine
- **dictionary（重点重构）**：从当前"搜索+全量树形表格"模式，重构为**左右分栏主从布局**
- **左侧面板**：字典名称搜索框 + 字典类型树（仅展示根节点/父级字典）+ 底部"新增子典"按钮
- **右侧面板**：标题栏 + "新增字典详情"按钮 + 子项平铺表格（列：字典标签、状态/el-switch、排序、备注、创建时间、操作/修改删除）
- **交互逻辑**：点击左侧树节点 → 右侧展示该节点的子项平铺列表；左树仅显示 parent_id 为空的根节点

### Monitor 模块（4页）

- **online**：hook 重命名 useRole→useOnline，实现真正的分页逻辑
- **login-log**：全面修正字段映射(ip→ipaddress, loginTime→createdTime)，移除后端不存在的 address/behavior 字段，新增 agent/loginType/description 列，修正搜索表单
- **operation-log**：全面修正字段映射(ip→ipaddress, username→creatorId, status→responseCode, operatingTime→createdTime)，移除 address/summary，新增 path/method/responseCode/responseResult(ReText)/statusCode/description 列
- **system-log**：新增 statusCode/description 列，修正搜索字段 requestTime→createdTime

### 封装组件最大化利用

- ReDialog (addDialog)：config/ip-rule 表单弹窗
- ReCol：所有表单两列栅格布局
- ReText：长文本列省略(description/value/email等)
- ReTreeLine：menu/dept 树形连接线
- usePublicHooks (switchStyle/tagStyle)：统一状态样式
- 公共常量(GenderChoices/IPRuleTypeChoices/DictStatusChoices)：替代硬编码选项

## Tech Stack

- **前端框架**: Vue3 + TypeScript（现有项目架构，保持不变）
- **UI 组件库**: Element Plus + Pure Admin 封装组件库
- **状态管理**: Vue Composition API (reactive/ref)
- **HTTP 请求**: Axios (@/utils/http)
- **日期处理**: dayjs
- **工具库**: @pureadmin/utils (handleTree/getKeyList/deviceDetection/isAllEmpty/cloneDeep)

## 实现方案

### 关键技术决策

1. **字典管理左右分栏重构（核心变更）**

当前字典页面是单一搜索表单 + handleTree 全量树形表格展示所有层级数据。需要重构为**左树右表主从模式**：

- 左侧面板（约25%宽度）：搜索框过滤字典名称 → el-tree 展示根节点字典（parent_id为空或'0'的记录）→ 节点文本格式为 `name(code)` → 底部"+ 新增子典"按钮用于新增根级字典类型
- 右侧面板（约75%宽度）：标题说明文字 → "+ 新增字典详情"按钮（蓝色描边风格）→ pure-table 平铺展示选中节点的子项（非树形），包含 label/status/sort/description/createdTime/操作 列
- 数据流：API返回扁平列表 → 前端分离根节点和子节点 → 左树绑定根节点数组 → 点击节点筛选子节点到右侧表格 → 新增子项时自动关联选中节点的id作为parentId
- hook.tsx 需要维护 `dictTypeList`（根节点）、`selectedDictType`（当前选中的字典类型ID）、`childDataList`（右侧子项数据）

2. **ReDialog 函数式表单迁移（config/ip-rule）**

遵循现有5个页面(user/role/menu/dept/dictionary)的成熟模式：

```
hook.tsx: openDialog() → addDialog({ contentRenderer, beforeSure })
form.vue: props接收 formInline, defineExpose暴露 getRef()
index.vue: 仅保留模板引用，移除 el-dialog 模板代码
```

3. **Monitor 字段映射修复（关键Bug修复）**

对照后端 log_router.py 的实际序列化字段逐一修正前端 columns prop 值：

- 登录日志：`ip`→`ipaddress`, `username`→移除, `address`→移除, `behavior`→移除, `loginTime`→`createdTime`, 新增`agent/loginType`
- 操作日志：`ip`→`ipaddress`, `username`→`creatorId`, `status`→`responseCode`, `operatingTime`→`createdTime`, `address/summary`→移除, 新增`path/method/body/responseResult/statusCode`

4. **Hook函数名修正**

Monitor 模块4个页面均误命名为 `useRole`，需分别重命名并同步修改 index.vue 的 import 引用。

5. **封装组件使用规范**

- 所有新增的状态列统一使用 `usePublicHooks()` 的 `tagStyle` 或 `switchStyle`
- 所有长文本列（description/value/email）使用 ReText 组件或 showOverflowTooltip
- 所有新增的时间列使用 dayjs 格式化为 `YYYY-MM-DD HH:mm:ss`
- 所有搜索表单的选项值使用 constants.ts 中定义的枚举常量

### 性能考虑

- 字典页面重构后，左侧树仅加载根节点，右侧按需加载子项，减少初始渲染量
- 表格新增列在分页控制下，性能影响可忽略
- addDialog 弹窗生命周期由 ReDialog 统一管理，内存释放更可靠
- monitor 日志字段修正不影响接口调用频率

## 实现注意事项

1. **字典重构注意事项**:

- API `/dictionary` POST 返回扁平列表，非树形结构；前端自行分离根节点(parent_id空)与子节点
- 左侧树节点点击时，通过 node.data.id 过滤 dataList 得到子项
- "新增子典"按钮：openDialog 时 parentId 设为 null/'0'
- "新增字典详情"按钮：openDialog 时 parentId 设为当前选中树节点的 id
- form.vue 需适配两种模式：有 parentId 时隐藏 name 字段，无 parentId 时显示 name
- 保持 PureTableBar 在右侧区域使用（列设置/密度切换/刷新/全屏）

2. **config/ip-rule 迁移注意事项**:

- config 使用直接导入的 getConfigList/configApi.create 等 API 函数
- ip-rule 也使用直接导入的 API 函数
- 各自保持原有调用方式不变
- addDialog contentRenderer 使用 h() 渲染 form.vue

3. **Monitor 字段修正注意事项**:

- 必须严格匹配后端 log_router.py 序列化的字段名（驼峰格式）
- 登录日志后端不返回 username/address/behavior，必须移除
- 操作日志后端不返回 username(address)/summary，必须移除
- 操作日志 status 列应改为 responseCode（HTTP响应码），使用 el-tag 根据 <400=success 否则=danger
- 时间列统一 formatter 格式 YYYY-MM-DD HH:mm:ss

4. **通用规范**:

- Hook 重命名时同步更新 index.vue 的 import 和解构赋值
- 所有新增列的 minWidth 参考 existing 列的宽度风格
- 代码注释使用中文

## 设计方案概述

本次优化涉及两类设计工作：(1) 字典管理页面的**全新左右分栏主从布局**；(2) 其余11个页面的**局部UI优化**（表单弹窗、表格列补充、样式统一）。重点描述字典管理的新布局设计。

### 字典管理 — 左右分栏主从布局

#### 页面整体结构

采用经典的**左树右表**Master-Detail模式，页面分为左右两个面板：

**左侧面板（字典类型树）**：

- 固定宽度约280px，带右边框分隔
- 顶部：紧凑型搜索框（输入框+清除按钮），用于快速过滤字典名称
- 主体：el-tree 组件展示字典类型树，节点文本格式为 `name(code)` 如"通知类型(notice_type)"、"状态(gender)"
- 底部："+ 新增子典"蓝色主按钮，用于新建顶级字典分类

**右侧面板（字典详情表）**：

- 自适应剩余宽度
- 顶部标题栏：左侧标题文字"字典管理（左侧字典树可通过右键单击进行修改和删除）"，右侧放置"+ 新增字典详情"描边按钮
- 主体：pure-table 平铺表格（非树形），展示选中字典类型的子项
- 底部：分页器（如需分页）

#### 交互流程

1. 页面加载 → 调用API获取全部字典数据 → 分离根节点和子节点 → 左树渲染根节点 → 默认选中第一个节点 → 右表展示其子项
2. 用户在左侧搜索框输入关键词 → 左树实时过滤节点
3. 用户点击左侧某节点 → 右表切换为该节点的子项数据
4. 用户点击"新增子典" → 弹出表单（parentId=null），填写字典名称等基本信息
5. 用户点击"新增字典详情" → 弹出表单（parentId=当前选中节点id），填写标签/值/排序等信息
6. 用户在右侧表格操作列点击"修改/删除" → 对应编辑/删除子项

### 其余页面 — 局部优化

其余11个页面保持现有布局不变，仅在以下方面做局部优化调整：

- config/ip-rule：将内联 el-dialog 替换为 addDialog 弹窗，提升交互一致性
- 所有表格：补充缺失的数据列，让信息更完整
- 所有状态列：统一样式（tagStyle/switchStyle）
- 所有表单：统一 ReCol 两列栅格布局

## Agent Extensions

### SubAgent

- **code-explorer**
- Purpose: 在实现过程中探索后端 DTO 和 ORM 模型，确认字段名映射关系（特别是 monitor 模块的登录日志/操作日志字段对应关系）
- Expected outcome: 确保前端 prop 值与后端序列化字段名完全一致，避免因字段名错误导致数据显示空白
