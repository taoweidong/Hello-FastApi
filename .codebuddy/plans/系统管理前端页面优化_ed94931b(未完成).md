## 产品概述

对系统管理模块（8个页面）和系统监控模块（4个页面）共12个前端页面进行全面优化，统一表单交互方式、修复字段映射错误、丰富表格数据展示、优化表单布局，最大化利用已封装组件。

## 核心功能

- 将 config 和 ip-rule 页面的 el-dialog 内联表单迁移为 ReDialog 函数式表单，与其他页面保持一致
- 修复 monitor 模块登录日志和操作日志与后端接口的严重字段不匹配问题（prop 值错误导致数据无法正确渲染）
- 为所有表格补充后端接口返回但前端未展示的字段列，让数据展示更完整
- 统一所有表单页面布局，使用 ReCol 两列栅格布局；role 表单从单列改为两列
- 修正 monitor 模块所有 hook 函数名误用 `useRole` 的问题
- 充分利用已封装组件：ReDialog、ReCol、ReText、usePublicHooks、公共常量等
- 实现 monitor 模块在线用户页面的分页功能（当前 handleSizeChange/handleCurrentChange 仅 console.log）

## 技术栈

- 前端框架：Vue3 + TypeScript（现有项目架构）
- 组件库：Element Plus + Pure Admin 封装组件
- 函数式弹窗：addDialog（@/components/ReDialog）
- 栅格布局：ReCol（封装 el-col）
- 文本省略：ReText
- 公共样式：usePublicHooks（switchStyle、tagStyle）
- 公共常量：GenderChoices、IPRuleTypeChoices、DictStatusChoices 等

## 实现方案

### 关键技术决策

1. **ReDialog 函数式表单迁移**：config 和 ip-rule 页面从 `el-dialog + dialogVisible + ruleForm` 模式迁移到 `addDialog + 独立 form.vue` 组件模式，遵循现有5个页面的成熟模式。

2. **Monitor 模块字段映射修复（关键）**：登录日志和操作日志的前端 columns 的 prop 值与后端 API 返回字段名不一致，导致数据无法正确显示。需对照后端 log_router.py 返回的字段名逐一修正。

3. **Hook 函数名修正**：monitor 模块3个日志页面和在线用户页面的 hook 函数均误命名为 `useRole`，需分别改为 `useOnline`、`useLoginLog`、`useOperationLog`、`useSystemLog`。

4. **表格列补全原则**：展示所有有业务价值的接口返回字段。长文本列使用 ReText 或 showOverflowTooltip。

5. **表单布局统一**：所有表单组件统一使用 `el-row + ReCol` 两列栅格布局。

### Monitor 模块后端字段映射（已验证）

#### 在线用户（stub 数据）

后端返回：`id, username, ip, address, system, browser, loginTime` — 前端已一致，无需修正

#### 登录日志

后端返回（log_router.py 实际序列化字段）：

- `id` ✓、`status` ✓、`ipaddress`（前端误写为 `ip`）、`browser` ✓、`system` ✓
- `agent`（前端缺失）、`loginType`（前端缺失）、`creatorId`（前端误展示为 `username`）
- `createdTime`（前端误写为 `loginTime`）、前端多余的 `address`、`behavior` 字段后端不存在

#### 操作日志

后端返回（log_router.py 实际序列化字段）：

- `id` ✓、`module` ✓、`path`（前端缺失）、`body`（前端缺失）、`method`（前端缺失）
- `ipaddress`（前端误写为 `ip`）、`browser` ✓、`system` ✓
- `responseCode`（前端缺失，前端误有 `status`）、`responseResult`（前端缺失）
- `statusCode`（前端缺失）、`creatorId`（前端误展示为 `username`）
- `createdTime`（前端误写为 `operatingTime`）、前端多余的 `address`、`summary` 字段后端不存在

#### 系统日志

后端返回字段与操作日志完全相同（共享 sys_logs 表），前端已基本正确，仅缺 `statusCode`、`description` 列

| ReText | user/dept/config/operation-log 表格 | 长文本列省略+Tooltip |

| renderMenuTypeTag | permission 权限树 | 菜单类型标签渲染 |


## 实现注意事项

- config 和 ip-rule 新建 form.vue 严格遵循现有模式（props FormProps + ref + getRef + defineExpose）
- addDialog 的 contentRenderer 使用 `() => h(editForm, { ref: formRef, formInline: null })` 模式
- beforeSure 中通过 `formRef.value.getRef()` 获取表单实例进行 validate 校验
- 表格列 dayjs 格式化统一使用 `YYYY-MM-DD HH:mm:ss` 格式
- monitor 模块字段修正必须与后端 log_router.py 返回的实际字段名完全一致
- 登录日志搜索：移除后端不支持的 `username` 筛选，新增后端支持的 `loginType` 筛选
- 操作日志搜索：`operatingTime` 改为 `createdTime` 与后端 DTO 一致
- config 使用直接导入的 API 函数（getConfigList 等），ip-rule 也使用直接导入函数，改造时保持各自的调用方式不变
- 在线用户页面的 handleSizeChange/handleCurrentChange 需实现真正的分页逻辑
