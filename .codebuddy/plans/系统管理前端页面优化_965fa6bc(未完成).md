## 产品概述

对系统管理模块前端8个子页面进行全面优化，统一表单交互方式、丰富表格数据展示、优化表单布局。

## 核心功能

- 将 config（系统配置）和 ip-rule（IP规则）页面的 el-dialog 内联表单改为 ReDialog 函数式表单（addDialog），与 user/role/menu/dept/dictionary 页面保持一致
- 为所有表格补充后端接口返回但前端未展示的字段列（如 email、description、updatedTime 等），让数据展示更完整
- 统一所有表单页面布局，使用 ReCol 组件实现两列栅格布局，提升表单视觉一致性和空间利用率
- 为 role 表单从单列布局改为两列 ReCol 布局，与 user/dept/dictionary 等页面风格统一
- 为 config 和 ip-rule 抽取独立 form.vue 表单组件，遵循现有 form 组件模式（props + getRef + defineExpose）

## 技术栈

- 前端框架：Vue3 + TypeScript（现有项目架构）
- 组件库：Element Plus + Pure Admin 封装组件（ReDialog、ReCol、PureTableBar 等）
- 样式方案：Tailwind CSS + SCSS
- 函数式弹窗：addDialog / closeDialog / updateDialog（@/components/ReDialog）

## 实现方案

### 整体策略

按页面逐个优化，每个页面完成三项改造：1) 表单改为 ReDialog；2) 表格列补全；3) 表单布局统一。优先改造公共基础设施，再逐个页面推进。

### 关键技术决策

1. **ReDialog 函数式表单**：config 和 ip-rule 页面从 el-dialog + dialogVisible 模式迁移到 addDialog + 独立 form.vue 组件模式。遵循现有 user/role/menu/dept/dictionary 页面的成熟模式：hook.tsx 中调用 addDialog 传入 contentRenderer 渲染表单组件，表单组件通过 props 接收 formInline 数据，通过 defineExpose 暴露 getRef 方法供 beforeSure 回调中做表单校验。

2. **表格列补全**：基于后端 ORM 模型字段对照前端 columns 定义，补充有业务价值且接口实际返回的字段。原则：展示所有有意义的字段，但隐藏过于冗长的字段（如 id 可隐藏，creatorId/modifierId 在无用户名解析时不展示）。updatedTime 统一添加；description 有值时展示；email/phone 等已有但未展示的字段补上。

3. **表单布局统一**：所有表单组件统一使用 `el-row + ReCol` 两列栅格布局（:value="12" :xs="24" :sm="24"），长文本字段（如描述、上级选择）使用全宽（默认 :value="24"）。role 表单从当前单列改为两列；config 和 ip-rule 新建 form.vue 时直接采用两列布局。

### 各页面具体优化细节

#### 用户管理（user）

- **表格列补全**：新增 email（邮箱）、description（描述）、updatedTime（更新时间）三列
- **表单**：已使用 ReDialog + 两列布局，无需改造
- **搜索表单**：新增邮箱搜索条件

#### 角色管理（role）

- **表格列补全**：新增 updatedTime（更新时间）列
- **表单布局优化**：从当前单列布局改为 ReCol 两列布局（name/code 一行，description 独占一行）
- **表单字段补全**：添加 isActive 状态开关字段

#### 菜单管理（menu）

- **表格列补全**：新增 isActive（状态）列，用 el-switch 展示；新增 updatedTime（更新时间）列
- **表单**：已使用 ReDialog + 两列布局，无需改造

#### 权限管理（permission）

- **表格列补全**：角色列表新增 description（描述）列
- **表单**：该页面无新增/编辑表单，无需改造

#### 系统配置管理（config）- 重点改造

- **表单改为 ReDialog**：移除 el-dialog + dialogVisible + ruleForm 模式，改为 addDialog + 独立 form.vue 组件
- **新建 form.vue**：使用 ReCol 两列布局，字段包含 key、value（textarea）、access（input-number）、isActive（switch）、inherit（switch）、description（textarea 全宽）
- **hook.tsx 重构**：openDialog 改为 addDialog 调用，handleSubmit 逻辑移入 beforeSure 回调；移除 dialogVisible/dialogTitle/ruleForm 状态
- **index.vue 精简**：移除 el-dialog 模板代码
- **表格列补全**：新增 updatedTime（更新时间）、description（描述）列
- **表格列优化**：id 列隐藏，value 列加 showOverflowTooltip

#### 部门管理（dept）

- **表格列补全**：新增 code（部门编码）、principal（负责人）、phone（手机号）、email（邮箱）、updatedTime（更新时间）列
- **表单**：已使用 ReDialog + 两列布局，无需改造

#### 字典管理（dictionary）

- **表格列补全**：新增 updatedTime（更新时间）列
- **表单**：已使用 ReDialog + 两列布局，无需改造

#### IP规则管理（ip-rule）- 重点改造

- **表单改为 ReDialog**：移除 el-dialog + dialogVisible + ruleForm 模式，改为 addDialog + 独立 form.vue 组件
- **新建 form.vue**：使用 ReCol 两列布局，字段包含 ipAddress、ruleType（select）、reason（input）、isActive（switch）、expiresAt（date-picker）、description（textarea 全宽）
- **hook.tsx 重构**：openDialog 改为 addDialog 调用，handleSubmit 逻辑移入 beforeSure 回调；移除 dialogVisible/dialogTitle/ruleForm 状态
- **index.vue 精简**：移除 el-dialog 模板代码、移除批量删除和清空按钮的确认提示优化
- **表格列补全**：新增 description（描述）列；createdAt 列统一改为 createdTime 并格式化为 dayjs；新增 updatedTime（更新时间）列
- **表格列优化**：状态列从 el-switch 改为 el-tag 展示（与 config 页面一致），避免误触；过期时间列格式化

### 性能与可靠性

- 表格新增列对渲染性能无明显影响，数据量在分页控制下
- addDialog 模式下弹窗生命周期由 ReDialog 统一管理，内存释放更可靠
- 表单校验在 beforeSure 回调中完成，校验失败不关闭弹窗，避免数据丢失

### 实现注意事项

- config 和 ip-rule 新建 form.vue 时，严格遵循现有表单组件模式（props FormProps 接口 + ref + getRef + defineExpose）
- addDialog 的 contentRenderer 使用 `() => h(editForm, { ref: formRef, formInline: null })` 模式
- beforeSure 中通过 `formRef.value.getRef()` 获取表单实例进行 validate 校验
- 表格列的 dayjs 格式化统一使用 `YYYY-MM-DD HH:mm:ss` 格式
- 新增的 updatedTime 列使用 `formatter` 而非 `cellRenderer`，与现有 createdTime 列保持一致
- config 和 ip-rule 的 API 调用方式不同（直接导入函数 vs 类实例方法），改造时保持各自的调用方式不变

## 架构设计

### 改造前后对比

**改造前（config/ip-rule）**：

```
index.vue → hook.tsx (dialogVisible + ruleForm) → index.vue (el-dialog 内联表单)
```

**改造后（config/ip-rule，与 user/role 等一致）**：

```
index.vue → hook.tsx (addDialog) → form.vue (独立表单组件)
```

## 设计风格

基于现有 Pure Admin 系统的企业级管理后台设计风格，保持专业、简洁的视觉体验。优化重点在于统一性和信息密度提升。

## 页面区块设计

### 搜索区域

- 统一水平排列的 inline 表单，bg-bg_color 背景色
- 表单控件宽度统一 w-45!
- 搜索/重置按钮图标和样式与现有一致

### 表格区域

- 表头背景色 var(--el-fill-color-light)，文字色 var(--el-text-color-primary)
- 新增列使用 dayjs formatter 或 el-tag/el-switch 渲染
- 状态列统一用 el-switch（可交互）或 el-tag（只读展示）
- 时间列统一 YYYY-MM-DD HH:mm:ss 格式

### 弹窗表单

- 统一使用 ReDialog 函数式调用
- 表单内使用 el-row + ReCol 两列栅格布局
- 短字段（input/select/switch）占半宽（:value="12"）
- 长字段（textarea/cascader）占全宽（默认 :value="24"）
- label-width 统一 82px
