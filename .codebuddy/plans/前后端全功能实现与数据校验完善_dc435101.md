## 需求概述

完善系统管理模块的前后端对接，实现所有页面的完整增删改查功能。

## 核心功能

### 1. 用户管理

- 用户列表查询（已实现）
- 新增用户：username/password 为必填项，前端阻止空值提交
- 编辑用户、删除用户、批量删除
- 重置密码、修改状态、分配角色

### 2. 角色管理

- 角色列表查询（已实现）
- 新增、编辑、删除角色
- 角色状态切换
- 分配菜单权限

### 3. 部门管理

- 部门列表查询（已实现）
- 新增、编辑、删除部门

### 4. 日志管理

- 登录日志/操作日志/系统日志查询（已实现）
- 批量删除日志
- 清空所有日志

### 5. 在线用户

- 强制下线功能

## 问题根因

前端 API 层（`system.ts`）只有查询接口，业务逻辑层（各 `hook.tsx`）的增删改操作均为占位代码，未调用真实后端 API。后端接口已完整实现，但部分 DTO 需完善空字符串处理。

## 技术栈

- **后端**: FastAPI + SQLModel + SQLAlchemy 2.0 (AsyncIO)
- **前端**: Vue 3 + TypeScript + Element Plus + Pure Admin
- **HTTP 客户端**: Axios (封装于 `@/utils/http`)

## 实现方案

### 方案概述

1. **前端 API 层完善**：在 `system.ts` 中补充所有增删改接口定义
2. **前端业务逻辑实现**：在各 `hook.tsx` 文件中实现真实 API 调用
3. **后端 DTO 优化**：确保所有 DTO 的 `field_validator` 正确处理空字符串

### 关键技术点

1. **API 接口命名规范**

- 创建：`POST /xxx/create`
- 更新：`PUT /xxx/{id}`
- 删除：`DELETE /xxx/{id}`
- 批量删除：`POST /xxx/batch-delete`
- 状态切换：`PUT /xxx/{id}/status`

2. **空字符串处理**

```python
@field_validator('field_name', mode='before')
@classmethod
def empty_str_to_none(cls, v):
if v == '' or v is None:
return None
return v
```

3. **前端表单提交验证**

- Element Plus 表单验证阻止空值提交
- `beforeSure` 回调中先验证表单再调用 API
